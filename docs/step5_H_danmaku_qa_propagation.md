# 子任务 H：弹幕空间中的“提问-解答”微观社区生态模型

## 1. 使用数据
* `danmaku/*.csv`（核心要素：`user_hash`, `content`, `progress`）

## 2. 数据处理指引 (Agent 自主选择路径)
* **问答对识别（QA Extraction）**：
  * 识别 Q（提问）：使用关键词规则（怎么、求问、不懂）或简单的文本分类器判定。
  * 圈定候选 A（回答）：设定物理时间窗口（如 Q 出现后的 15 秒画面内）和现实时间窗口。
  * 确认匹配：计算候选 A 与 Q 的语义相似度，或识别特有的回复句式（如“前面那个xxx，是因为…”），完成 $(Q_i, A_i)$ 的组对。
* **计算指标**：统计每个视频的“提问响应率（Response Rate）”和“平均响应潜伏期（Latency）”。

## 3. 分析与验证分支逻辑
* **检验假设**：弹幕生态中存在自发的互助传播网络，且不同标签分区（如“AI编程” vs “AI搞笑”）的知识响应率有显著差异。
* **分支 A（捕获到高频有效的 QA 对）**：响应率可观（如 > 10%），且响应时长短。-> 弹幕在 B 站不仅是情绪工具，更是高效的分布式知识注脚系统。
* **分支 B（QA 匹配率极低或大量噪音）**：弹幕中充斥问题但无人作答，或者回答完全风马牛不相及。-> 弹幕的异步特性决定了它不适合承载结构化互助。

## 4. 业务洞察推演
* **基于分支 A（繁荣的互助生态）**：向平台建议开发“弹幕高亮追问功能”或“问题悬赏弹幕”，将这种自发的互助行为产品化，提升学习社区的壁垒。
* **基于分支 B（信息孤岛）**：指导 UP 主不要指望观众自治。UP 主应定期挖掘视频弹幕中的“未解答高频 Q”，作为下一期视频的选题来源（用户洞察反哺内容创作）。

## 5. 局限性与未来深化方向
* **追踪断链问题**：弹幕的 `user_hash` 只是该视频内的唯一标识，无法追踪一个“热心解答者”是否在全网各个 AI 视频中都在解答（专家用户识别）。未来若能获取全局用户 ID，可通过构建“用户-用户”多层异质图神经网络（Heterogeneous Graph Neural Networks），精准挖掘出社区内部的“隐形课代表/知识枢纽”。
