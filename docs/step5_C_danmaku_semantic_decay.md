# 子任务 C：弹幕生命周期与受众泛化（语义衰减分析）

## 1. 使用数据
* 指定的爆款视频子集及其 `danmaku/*.csv`（需包含数十万级弹幕规模）。

## 2. 数据处理指引 (Agent 自主选择路径)
* **切分依据抉择**：
  * 路径 1（基于时间）：按弹幕发送的现实时间 `ctime` 将弹幕切分为“首日/首周”（核心粉）与“长尾期”（泛受众）。
  * 路径 2（基于播放进度）：按视频的 `progress`，观察视频开头到结尾的语义密度。
* **语义量化工具**：不要硬编码词典。使用 TF-IDF 或 TextRank 自动提取早期弹幕的 Top 100 关键词（通常为专业词），再观察这 100 个词在后期弹幕总词汇中的占比曲线。

## 3. 分析与验证分支逻辑
* **检验假设**：随着时间推移，弹幕的信息熵（或专业词密度）呈下降趋势，无意义互动词（如“2333”）上升。使用 Mann-Whitney U 检验对比两阶段分布。
* **分支 A（显著衰减）**：证实了“破圈即稀释”的受众泛化规律。
* **分支 B（未见衰减甚至反弹）**：后期依然保持高浓度的专业讨论。-> 视频触及了长尾的“硬核搜索引擎流量”（用户通过搜索特定技术问题进来，而非首页推荐）。

## 4. 业务洞察推演
* **若发现显著衰减**：向平台提出建议：当视频播放量突破阈值引发受众泛化时，推荐系统应引入“弹幕折叠/降噪机制”，保护核心圈层的讨论氛围不被破坏。
* **若发现长尾硬核流量**：指导教程类 UP 主注重视频简介的 SEO 优化，因为几个月后的长尾搜索流量同样具备极高的商业变现价值（精准受众）。

## 5. 局限性与未来深化方向
* **圈层黑话的漂移**：当前方法可能把泛化人群创造的新梗（如“赛博朋克”）误判为低质量词汇。未来可通过动态词嵌入（Dynamic Word Embeddings）追踪同一个词在视频生命周期内语义空间位置的转移。
