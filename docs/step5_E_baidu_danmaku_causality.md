# 子任务 E：百度热点与 B 站弹幕热词的时序因果检验

## 1. 使用数据
* 百度指数md "D:\Claude_Code\bilibili-ai-tag-analysis\results\baidu_index_event_report.md" + 所有视频的弹幕数据（按 `ctime` 日期聚合）。

## 2. 数据处理指引 (Agent 自主选择路径)
* **数据对齐与插值**：处理节假日或断更导致的数据缺失（如线性插值）。对两条序列进行 Min-Max 归一化。
* **平稳性校验（极其重要）**：必须先对时间序列进行 ADF 检验（Augmented Dickey-Fuller test）。如果不平稳，必须进行一阶或二阶差分直至平稳，否则后续因果分析全是“伪回归”。

## 3. 分析与验证分支逻辑
* **检验假设**：使用互相关函数（CCF）和格兰杰因果检验（Granger Causality Test）寻找两条曲线的领先-滞后关系（Lag）。
* **分支 A（百度单向领先 B 站）**：百度指数突增是 B 站弹幕讨论量增加的格兰杰原因，Lag 通常为 1-3 天。
* **分支 B（B 站单向领先百度）**：B 站弹幕热词突增是百度指数上升的格兰杰原因。
* **分支 C（双向因果或瞬时同步）**：两者同时爆发，Lag = 0。

## 4. 业务洞察推演
* **若为分支 A（大众吃瓜模式）**：说明 AI 话题是由外部大新闻引爆，用户先在百度搜索，随后在 B 站寻找深度解说视频发弹幕。UP 主需要建立全网热点监控，获得 1-2 天的“制作窗口期”。
* **若为分支 B（极客造浪模式）**：说明 B 站科技区是中文互联网 AI 新趋势的“发源地”，硬核圈层的讨论外溢最终引发了全网搜索。平台应加大对这些源头极客 UP 主的资源倾斜。

## 5. 局限性与未来深化方向
* **格兰杰因果不等于实质因果**：这只是一种时间预测上的统计关系。未来应考虑加入微博热搜、知乎热榜作为中间内生变量，构建向量自回归模型（VAR）或脉冲响应函数（IRF），描绘完整的多平台信息传染网络。
