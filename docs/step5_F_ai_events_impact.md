# 子任务 F：重大 AI 事件对社会心理（弹幕情感）的冲击效应

## 1. 使用数据
* 外部人工整理的重大 AI 事件时间表（如 ChatGPT-4o 发布，DeepSeek R1 开源， 参考百度指数"D:\Claude_Code\bilibili-ai-tag-analysis\results\baidu_index_event_report.md"）。
* 聚合的日级弹幕情绪指标 / “焦虑词”词频。

## 2. 数据处理指引 (Agent 自主选择路径)
* **事件研究法框架 (Event Study)**：
  * 定义事件日 $T_0$。
  * 设定估计窗口（如 $[-30, -10]$ 天）拟合正常情绪基线模型。
  * 设定事件窗口（如 $[-2, +7]$ 天）计算“异常情绪波动值（Abnormal Sentiment Return）”。
* **排除混淆**：若两个大事件间隔过近（重叠），需剔除或合并分析。

## 3. 分析与验证分支逻辑
* **检验假设**：技术突破型事件会在短期内引起 B 站 AI 相关视频弹幕的焦虑词密度显著上升。
* **分支 A（显著冲击且快速衰减）**：事件发生后 2-3 天焦虑值达到峰值，一周后恢复基线。
* **分支 B（阶梯型永久偏移）**：某次划时代事件（如 Sora）后，社区的基础焦虑值永久上升了一个台阶。
* **分支 C（毫无显著冲击）**：事件无法引起统计学上的情感异动。

## 4. 业务洞察推演
* **若证实分支 A/B**：证实了“技术焦虑的脉冲式爆发规律”。这为平台提供了一套**内容对冲策略**的理论依据（当技术大爆炸发生时，平台算法不应只推技术解析，而应同步放量推送“如何使用 AI 变现/提效”的实用化解视频以抚平焦虑）。
* **若证实分支 C（技术脱敏）**：说明用户对“颠覆性 AI”的宏大叙事已产生免疫。指导 UP 主放弃空泛的吹捧/贩卖焦虑，回归到微观、实用的“手把手落地场景”教学。

## 5. 局限性与未来深化方向
* **情感维度的单一性**：单纯的积极/消极或“焦虑词频”过于粗糙。未来需引入细粒度的情绪图谱模型（如基于 Ekman 六大基本情绪），探究冲击究竟是引发了“恐惧”、“惊奇”还是“愤怒”。
