本阶段是数据采集的第一步。你的首要任务是获取指定AI标签在百度指数上的搜索热度趋势，识别出明显的热度峰值，并结合网络搜索（News/Web Search）找出导致这些峰值的具体现实事件。最后，你需要生成相关的可视化图表和一份分析报告。

**百度数据已获取**
2. **B站 API 预研与测试准备**：阅读 `https://github.com/nemo2011/bilibili-api` 文档，梳理后续 B站数据采集所需的 API 接口使用规范、异步并发逻辑与异常处理机制，采取“先测试、后运行”的原则，最终产出一份详细的 API 使用调研报告。

## 二、 具体执行步骤

### 2.1 百度热点数据获取
* **目标关键词**：人工智能、AI、ChatGPT、Sora、DeepSeek、机器学习、AI焦虑。
* **时间窗口**：2024年1月1日 —— 2026年1月31日。
* **数据粒度**：日级（Daily）。
* **执行动作**：
    * 编写Python代码（建议保存在 `src/spiders/baidu_index_scraper.py`），利用合适的爬虫库或第三方接口获取上述关键词的百度指数。小心反爬虫
    * 将获取到的原始数据保存为CSV文件，存放到 `data/raw/baidu_index_2401_2601.csv`。

### 2.2 峰值识别与新闻事件归因
* **执行动作**：
    * 基于获取到的时间序列数据，使用适当的统计算法（如Z-Score、移动平均差值等），自动识别出每个关键词在2024.01-2026.01期间的**显著热度峰值（Peaks）**。和其热度持续时间
    * 对于每一个识别出的峰值（需精确到具体日期或正负3天的时间段），你必须执行**网络检索**，查找该时间段内发生的重大AI相关新闻或事件。
    * 记录事件的起因（为什么会产生这个峰值）以及权威新闻的链接（URLs）。

### 2.3 数据可视化
* **执行动作**：
    * 编写Python脚本（建议保存在 `src/analysis/plot_trends.py` 或 Jupyter Notebook 中）。
    * 绘制各关键词的时间序列折线图。
    * **关键要求**：必须在折线图的峰值处添加**文本标注（Annotations）**，简短说明触发该峰值的关键事件（例如：“Sora发布”、“DeepSeek R1开源”、“WAIC大会”等）。
    * 将生成的图表以PNG或PDF格式保存至 `results/figures/` 目录下（如 `results/figures/baidu_trend_with_events.png`）。

### 2.4 生成阶段性报告
* **执行动作**：
    * 综合上述数据、峰值原因和可视化图表，撰写一份Markdown格式的分析报告。
    * 报告必须包含：时间序列总体趋势描述、核心峰值事件梳理表（包含：日期、持续时间，涉及标签、峰值数值、事件原因、新闻链接）、以及对应的可视化图表。
    * 将报告保存至 `results/baidu_index_event_report.md`。


### 模块二：B站 API 文档阅读与测试预研
#### 2.1 文档阅读与逻辑梳理
* **执行动作**：
    * 访问并仔细阅读 GitHub 仓库：https://github.com/Nemo2011/bilibili-api/tree/main 及其相关文档。
    * 重点提取以下技术规范：
        * **异步编程模式**：如何正确使用 `asyncio` 调用相关接口。
        * **异常与风控处理**：梳理 `bilibili_api.exceptions` 中的常见错误（特别是网络超时、频率限制-799、Cookie失效等），并设计重试退避（Exponential Backoff）逻辑。
        * **凭证配置**：了解如何配置 `Credential` 类（SESSDATA, bili_jct, buvid3 等）。

#### 2.2 核心数据需求与 API 映射匹配
* **项目数据需求清单**（必须梳理出对应的具体调用方法或请求端点）：
    1. **视频搜索与列表获取**：按标签和时间范围搜索，提取 BVID 列表。
    2. **视频基础信息**：根据 BVID 获取播放量(view)、弹幕数(danmaku)、点赞/投币/收藏数、发布时间(pubdate)、UP主ID(owner.mid)、CID 等。
    3. **视频标签提取**：获取视频挂载的具体标签（新接口或旧接口）。
    4. **弹幕数据获取**：根据 CID 获取视频弹幕XML（包含弹幕内容、时间戳、用户Hash等）。
    5. **UP主信息获取**：根据 UP主ID(mid) 获取该UP主的当前粉丝数(fans)。

#### 2.3 测试代码编写与 API 调研报告产出
* **执行动作**：
    * **先测试再运行**：在 `src/spiders/` 目录下创建一个测试脚本 `test_bili_api.py`，使用硬编码的单条数据（如指定一个具体的 BVID 和 MID）对上述接口进行**不带循环的单次异步请求测试**。
    * **撰写调研报告**：整理文档阅读心得、API映射表、异常处理策略以及测试运行的结果，撰写一份详细的 API 使用报告。
    * 报告必须包含：需要获取的数据清单及其对应的 `bilibili-api` 函数代码示例、异常处理最佳实践（try-except块示例）。
    * 将报告保存至 `results/api-report.md`。
