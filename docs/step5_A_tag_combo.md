# 子任务 A：标签组合的“爆款效率”分析与挖掘

## 1. 使用数据
* `cleaned-data/bilibili_video_cleaned.csv`（核心字段：`bvid`, `tags`, `stat_view`, `owner_fans`）

## 2. 数据处理指引 (Agent 自主选择路径)
* **标签预处理**：将 `tags` 拆分为 List。需要对长尾标签进行低频截断（例如过滤掉全局出现次数 < 10 的标签，避免稀疏矩阵爆内存）。
* **目标变量构建**：由于播放量呈长尾分布，绝对不应直接使用平均值。建议构建**相对爆款率 (Relative Performance)** = `单视频播放量 / 该UP主历史中位数播放量`（如果数据不足，可用 `单视频播放量 / 同粉丝量级UP主的中位数播放量` 代替）。
* **挖掘算法选择**：
  * 若计算资源充足：使用 FP-Growth 挖掘频繁项集（速度快于 Apriori）。
  * 若标签维度极大：构建共现矩阵（Co-occurrence Matrix），提取网络图中的最大连通子图（Community Detection）来识别自然形成的“标签簇”。

## 3. 分析与验证分支逻辑
* **检验假设**：特定的多标签组合（如 宽泛+垂直）其“相对爆款率”显著高于单一标签。
* **分支 A（数据高度显著）**：发现某些特定组合（如 A+B+C）的 Lift（提升度）> 1.5，且 p-value < 0.05。 -> 提取这些组合及其特征（是跨界组合还是深度垂直？）。
* **分支 B（无显著差异/独立性强）**：发现组合的期望表现几乎等于各标签单独表现的乘积（Lift ≈ 1），或毫无规律。 -> 说明算法是按单标签独立召回的，不存在“魔法组合”。
* **分支 C（负向显著）**：发现某些标签组合 Lift < 1。 -> 存在相互排斥的“毒药组合”（如“娱乐”+“硬核源码”导致两边受众都不买账）。

## 4. 业务洞察推演 (基于分支结果)
* **若进入分支 A**：为 UP 主生成“标签搭配公式表”，指导冷启动期如何通过“热门大标签导流 + 精准小标签固粉”的组合策略吃透推荐算法。
* **若进入分支 B/C**：警告 UP 主放弃“玄学堆标签”的幻想，指出盲目堆砌不相关热门标签会触发算法的降权惩罚（完播率惩罚），建议精简标签数量。

## 5. 局限性与未来深化方向
* **若当前特征不足**：标签往往和视频的“真实分区(tname)”绑定。未来方向应引入层次线性模型（HLM），将分区作为宏观层，标签作为微观层，探究“跨区标签”的异质性效应（例如科技区UP主用了搞笑标签的效果）。
