# 第3步：数据预处理前的探索性分析 (Phase 2 - Pre-clean EDA)

## 一、 任务概述
本阶段的任务是接手已经采集完成的 B站视频基础数据集（6393+条记录），进行数据第一波探索性数据分析（EDA）。
你的核心目标是摸清数据的底数：了解各个标签的真实分布情况，提取高频标签，并掌握视频播放量的层级结构。

## 二、 数据来源
* **视频基础数据**：`data/raw/bilibili_video_info.csv`
* **目标字段**：`bvid`, `tags`, `search_tag`, `stat_view`

## 三、 具体执行步骤


### 3.2 EDA 1：视频标签统计与分布
基于清洗前的数据，进行以下维度的标签统计：
1. **按检索源统计 (search_tag)**：
   * 统计每个 `search_tag` 目录下实际采集到的去重视频数量，绘制柱状图。
2. **视频真实标签统计 (tags)**：
   * 读取视频自带的 `tags` 字段（数据以 `|` 或逗号分隔）。
   * 将所有视频的标签进行拆分、平铺（Explode），统计全量数据中所有独立标签的出现频次。
   * **提取 Top 20**：筛选出出现频次最高的 20 个标签。
   * **聚合 Others**：将排名 21 及以后的所有标签频次进行求和，统一归为 "Others"（其他）。
   * **可视化**：绘制 Top 20 标签 + Others 的频次分布图（推荐使用横向条形图或饼图，需标注具体数值），保存至 `results/figures/tag_distribution_top20.png`。

### 3.3 EDA 2：播放量阶梯分布分析
将视频的 `stat_view`（播放量）进行分箱（Binning）统计，洞察流量的层级分布。
1. **分箱规则**：
   * 级别 1：`< 50,000`（< 5万）
   * 级别 2：`50,000 - 200,000`（5万 - 20万）
   * 级别 3：`200,000 - 1,000,000`（20万 - 100万）
   * 级别 4：`> 1,000,000`（大于 100万）
2. **统计执行**：计算每个播放量区间内的视频数量及占比。
   * *⚠️ Agent 注意*：由于采集规则设定了 `播放量 ≥ 50,000` 的硬性过滤条件，理论上 `< 5万` 这个区间的数据量应为 0 或极少。请如实统计，如果存在数据，请在报告中标记为“异常漏网/早期规则变动数据”。
3. **可视化**：绘制播放量分布的环形图或柱状图，保存至 `results/figures/view_distribution_bins.png`。
