# 第2步：B站视频与弹幕核心数据采集 (Phase 1 - Part 2)

## 一、 任务概述
在完成 B站 API 的预研与测试后，本阶段将正式启动大规模的数据采集。你将针对预设的标签体系，在指定的时间窗口内，按照严格的比例采样策略和数据清洗下限，获取视频基础信息、标签列表以及对应的弹幕数据，为后续深度分析提供高质量的数据源。

## 二、 采集参数与标准定义

### 2.1 目标标签体系 (Tags)
针对以下每个标签，**至少采集 1000 个视频**（确保最终样本的统计显著性）：
* **主标签**：人工智能
* **宽泛标签**：科技
* **相似标签**：机器学习、AI
* **窄标签/具体产品**：ChatGPT、Sora、DeepSeek
* **情感/社会化标签**：AI焦虑
* **细分场景标签**：AI绘画、AI编程 

### 2.2 时间窗口
* **开始日期**：2024年1月1日
* **结束日期**：2025年12月31日

### 2.3 核心筛选下限 (硬性过滤标准)
为保证弹幕内容的丰富度及分析的有效性，所有入库的视频必须**同时满足**以下条件：
* **播放量 (View)** $\ge$ 100,000
* **弹幕量 (Danmaku)** $\ge$ 200

### 2.4 按比例采样策略 (Proportional Sampling)
* **禁止均匀抓取**：不能每天固定抓取相同数量的视频。
* **执行逻辑**：
  1. 优先评估或获取该标签在所选时间段内的每日视频发布分布量（可通过搜索接口的页数或粗略统计评估）。
  2. 视频发布数量多的日期，分配更多的采集配额（按比例多采集）；发布少的热度低谷期，分配较少的配额。
  3. 最终汇总确保每个标签下的有效视频总量达到或超过 1000 个。（一个视频两个标签，每个标签视频数量加一）
  4. 总是优先抓取Top5 播放量视频和top5弹幕数视频。（重复值在抓取后去除）

### 3.2 弹幕数据获取
* 针对上述成功获取基础信息的视频，根据其 `cid` 获取弹幕数据。
* **数据落盘**：为了避免单文件过大，请在 `data/raw/danmaku/` 目录下，以 `BVID.csv` 的命名方式（例如 `data/raw/danmaku/BV1xx411c7mD.csv`），单独存放每个视频的弹幕。
* **弹幕采集字段**：
  * `danmaku_id` (弹幕唯一ID)
  * `content` (弹幕文本内容)
  * `progress` (在视频中出现的时间节点)
  * `ctime` (弹幕发送的现实时间戳)
  * `user_hash` (发送者的隐匿哈希值)

## 四、 Agent 执行规范与风控限制

1. **异步并发与频率控制**：必须使用异步框架执行。严格限制并发数，并在每次接口请求间加入随机的异步休眠（`asyncio.sleep(x秒)`）。
2. **异常接管**：
   * 遇到 `-799` 请求过频错误时，必须执行指数退避（Exponential Backoff），强制休眠至少 30 秒后再重试。
   * 对于被删除或权限不足（例如要求大会员）导致获取失败的视频，记录日志并直接跳过，不要中断整体循环。
3. **日志记录**：在控制台输出详细的进度日志，例如：“[标签: ChatGPT] 当前已采集 450/1000 | 遇到风控，休眠等待中...”
4. **暂停与等待**：所有指定标签的数据拉取完毕并成功写入 CSV 文件后，**必须停止当前工作流程**。
5. **回复标准**：在终端/对话框中输出：“已完成所有指定标签的B站视频与弹幕核心数据采集任务。视频基础数据已存入 `data/raw/bilibili_video_info.csv`，弹幕详情已分类存入 `data/raw/danmaku/` 目录。等待下一步数据清洗与UP主信息补充的指示。”
