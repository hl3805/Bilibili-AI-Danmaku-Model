# 子任务 D：弹幕情绪时间线与视频结构的映射

## 1. 使用数据
* `danmaku/*.csv`（重点依赖 `progress`, `content`）

## 2. 数据处理指引 (Agent 自主选择路径)
* **时间序列重构**：将视频按长度归一化（0% - 100%），设定 2% 为一个滑动窗口（Sliding Window）。
* **情感打分容错**：开源模型（如 SnowNLP）对 B 站弹幕效果极差。建议提取少量样本，用 LLM 辅助构建一个“B站特色情感修饰词表”（如：牛逼、离谱、就这），结合词典法与预训练模型打分，计算每个窗口的平均极性。
* **峰值检测**：使用 `scipy.signal.find_peaks` 提取情绪和弹幕密度的局部极大值点。

## 3. 分析与验证分支逻辑
* **检验假设**：同类型视频（如 AI 教程类或 AI 资讯类）在时间线上具有高度相似的情绪起伏结构（如动态时间规整 DTW 距离显著小于随机序列）。
* **分支 A（发现普遍的“黄金节奏”模型）**：比如所有高赞视频都在 15% 处有悬念小高峰，在 85% 处有密集的情绪共鸣点。
* **分支 B（无通用规律，完全随机或平坦）**：说明内容驱动本身极其多样，不存在所谓“三分钟必须有一高潮”的刻板规律。

## 4. 业务洞察推演
* **基于分支 A 的赋能**：提炼出对应品类视频的“标准剧本心电图”，指导中腰部 UP 主按此节奏点铺设内容包袱和互动诱饵（如“懂了的扣1”应放在什么位置收益最高）。
* **基于分支 B 的反思**：警示 MCN 机构不要盲目套用工业化模版，在 AI 这样信息密度极高的垂类，平铺直叙的硬核讲解只要干货足，同样能获得极佳的长尾效应。

## 5. 局限性与未来深化方向
* **模态缺失**：仅凭进度条盲人摸象是不准的。未来方向必须引入多模态（Multi-modal）对齐，将音频的音量/语速特征（Prosody）和画面切换频率（Shot Boundary）与弹幕情绪峰值进行交叉相关性分析，才能知道到底是什么“画面/声音”触发了弹幕高潮。
