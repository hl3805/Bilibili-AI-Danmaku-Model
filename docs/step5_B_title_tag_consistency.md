# 子任务 B：标题-标签一致性对推荐机制的调节效应

## 1. 使用数据
* `cleaned-data/bilibili_video_cleaned.csv`

## 2. 数据处理指引 (Agent 自主选择路径)
* **文本向量化方案**：
  * 基础路径：使用 TF-IDF 提取标题关键词，与标签计算 Jaccard 相似度。
  * 进阶路径（推荐）：使用预训练语言模型（如 `Sentence-BERT` 或 `text2vec-chinese`）分别对标题和标签集合进行 Embedding，计算余弦相似度（Cosine Similarity），得出一致性连续变量 $S \in [0, 1]$。
* **控制变量设定**：必须纳入对数化的粉丝数 `log(owner_fans)`、视频时长、发布时间戳（转化为距今天数）作为回归的控制变量。

## 3. 分析与验证分支逻辑
* **检验假设**：标题与标签的一致性 $S$ 与播放量呈正相关（或倒U型关系）。建议使用 OLS 多元回归或广义相加模型（GAM）拟合。
* **分支 A（强正相关）**：$S$ 系数显著为正。-> 推荐算法在冷启动期极其依赖“文题相符”来锁定种子人群。
* **分支 B（倒 U 型显著）**：适度偏离（标题带点悬念，标签极其老实）播放量最高。
* **分支 C（无显著关系或负相关）**：“标题党”横行，只要标题足够吸引眼球（即便和标签偏离），高点击率足以弥补完播率的折损。

## 4. 业务洞察推演
* **针对算法逻辑**：如果验证了分支 A，说明 B 站推荐系统已具备强大的语义纠偏能力，“挂羊头卖狗肉”会被迅速限流。
* **针对内容运营**：如果验证了分支 B，指导 UP 主采取“标题负责情绪和悬念，标签负责 SEO（搜索引擎优化）和机器定性”的双轨制命名策略。

## 5. 局限性与未来深化方向
* **文本相似度的盲区**：纯 Embedding 无法识别“反讽”或“反差萌”式的优质标题。未来可利用 LLM 进行 Few-shot 意图打标（识别：疑问型、震惊型、教程型标题），在不同标题意图下再分别计算一致性的影响。
